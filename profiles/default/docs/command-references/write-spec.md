## Example Output

```markdown
# User Authentication with OAuth

## Overview
Enable users to authenticate using their existing Google or GitHub accounts,
eliminating the need for separate credentials and improving onboarding experience.

## User Stories
- As a new user, I want to sign up with Google, so that I can start using the app immediately
- As a returning user, I want to log in with GitHub, so that I don't need another password
- As a user, I want to log out, so that I can secure my account on shared devices

## Requirements

### Functional
- Support Google OAuth 2.0 authentication
- Support GitHub OAuth authentication
- Create user record on first OAuth login
- Link OAuth identity to existing user if email matches

### Non-Functional
- OAuth flow should complete in under 3 seconds
- Support mobile and desktop browsers
- Gracefully handle OAuth provider downtime

## Acceptance Criteria
- [ ] "Sign in with Google" button visible on login page
- [ ] "Sign in with GitHub" button visible on login page
- [ ] Successful OAuth redirects to dashboard
- [ ] Failed OAuth shows error message
- [ ] User profile shows connected OAuth providers
- [ ] Logout clears session and redirects to home

## Scope

### In-Scope
- Google OAuth integration
- GitHub OAuth integration
- Basic session management
- User profile with OAuth info

### Out-of-Scope
- Email/password authentication (future phase)
- Two-factor authentication (future phase)
- OAuth token refresh (handled by provider)

## Dependencies
- OAuth provider developer accounts (Google, GitHub)
- Existing User model

## Risks & Considerations
- OAuth provider API changes may require updates
- Rate limiting on OAuth endpoints during high traffic
```
