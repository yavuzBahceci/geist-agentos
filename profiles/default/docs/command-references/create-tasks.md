## Example Output

```markdown
# Tasks: User Authentication with OAuth

## Task Group 1: OAuth Provider Setup

- [ ] **1.1 Configure Google OAuth credentials**
      Set up Google OAuth application and store credentials
      
      **Acceptance Criteria:**
      - [ ] Google OAuth app created in Google Cloud Console
      - [ ] Client ID and secret stored in environment variables
      - [ ] Redirect URI configured correctly
      
      **Implementation Hints:**
      - Store in .env: GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET
      - Reference: existing env pattern in config/

- [ ] **1.2 Configure GitHub OAuth credentials**
      Set up GitHub OAuth application and store credentials
      
      **Acceptance Criteria:**
      - [ ] GitHub OAuth app created in GitHub settings
      - [ ] Client ID and secret stored in environment variables
      - [ ] Callback URL configured correctly

## Task Group 2: OAuth Flow Implementation

- [ ] **2.1 Create OAuth initiation endpoints**
      Create routes that redirect to OAuth providers
      
      **Acceptance Criteria:**
      - [ ] GET /auth/google redirects to Google OAuth
      - [ ] GET /auth/github redirects to GitHub OAuth
      - [ ] State parameter included for CSRF protection
      
      **Implementation Hints:**
      - Follow route pattern in routes/api/
      - Use existing middleware for error handling

- [ ] **2.2 Create OAuth callback handlers**
      Handle OAuth provider callbacks and create sessions
      
      **Acceptance Criteria:**
      - [ ] GET /auth/google/callback processes Google response
      - [ ] GET /auth/github/callback processes GitHub response
      - [ ] User created or found by email
      - [ ] Session created on success
      - [ ] Error page shown on failure

## Task Group 3: Session Management

- [ ] **3.1 Implement session middleware**
      Create middleware to manage user sessions
      
      **Acceptance Criteria:**
      - [ ] Session attached to request object
      - [ ] Session persists across requests
      - [ ] Session expires after configured time
      
      **Implementation Hints:**
      - Reference: existing middleware pattern in middleware/

- [ ] **3.2 Implement logout functionality**
      Allow users to end their session
      
      **Acceptance Criteria:**
      - [ ] POST /auth/logout clears session
      - [ ] User redirected to home page
      - [ ] Session cookie removed

## Task Group 4: UI Components

- [ ] **4.1 Add OAuth login buttons**
      Create login page with OAuth provider buttons
      
      **Acceptance Criteria:**
      - [ ] "Sign in with Google" button visible
      - [ ] "Sign in with GitHub" button visible
      - [ ] Buttons link to OAuth initiation endpoints
      - [ ] Loading state shown during redirect

- [ ] **4.2 Display user profile info**
      Show authenticated user's information
      
      **Acceptance Criteria:**
      - [ ] User name displayed in header
      - [ ] User avatar shown (from OAuth provider)
      - [ ] Logout button visible when logged in
```

| Tip | Description |

| **Order matters** | Foundation tasks (models, config) before features |
| **Include hints** | Reference existing patterns from basepoints |
| **Clear AC** | Each task needs testable acceptance criteria |
