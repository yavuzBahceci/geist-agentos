## Tips

| Tip | Description |
|-----|-------------|
| **Follow patterns** | Always reference basepoints for coding patterns |
| **Run validation often** | Catch issues early, not at the end |
| **Small commits** | Commit after each task for easy rollback |
| **Check AC** | Verify each acceptance criterion before marking done |

## When to Use

| Use `/implement-tasks` when... | Use `/orchestrate-tasks` when... |
|-------------------------------|----------------------------------|
| Single agent working alone | Multiple agents working in parallel |
| Small to medium task lists | Large task lists with many groups |
| Sequential implementation | Parallel implementation needed |
| Simple projects | Complex projects with many components |
